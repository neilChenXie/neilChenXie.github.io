# 博客标签自动生成说明

## 功能介绍
这个Jekyll插件会自动从所有文章中提取标签，并自动为每个标签生成一个汇聚页面。

## 工作原理

### 1. 插件文件：`_plugins/tag_generator.rb`
- 在每次构建时扫描所有文章中的 `tags` 字段
- 自动在 `_tags_page/` 目录下生成对应的标签页面文件
- 只会生成不存在的标签页面，已有的不会覆盖

### 2. 标签页面模板：`_layouts/page_tag.html`
- 自动根据标签名称查找所有包含该标签的文章
- 展示文章列表，包括标题、日期、摘要和标签

### 3. 配置：`_config.yml`
- 已启用 Jekyll 插件支持
- `_tags_page` 集合配置为输出模式

## 如何使用

### 步骤1：在文章中添加标签
在文章的 YAML 前置数据中添加 `tags` 字段：

```markdown
---
title: 文章标题
tags:
  - 标签1
  - 标签2
  - 标签3
---
```

### 步骤2：构建网站
运行以下命令构建网站：

```bash
bundle exec jekyll build
```

### 步骤3：验证生成
- 新标签对应的页面会自动在 `_tags_page/` 目录下生成
- 生成的页面会在构建时出现 `Tag Generator: Generated tag page for '标签名'` 提示
- 构建完成后，访问 `/tag/标签名/` 即可看到该标签的汇聚页面

## 标签汇聚页效果
每个标签的汇聚页会展示：
- 所有包含该标签的文章列表
- 文章标题（可点击进入详情）
- 发布日期
- 文章摘要（前50个单词）
- 文章的其他标签

## 特点
✅ 完全自动化 - 无需手动创建标签页面
✅ 智能检测 - 只为实际使用的标签生成页面
✅ 非侵入式 - 不会覆盖手动创建的页面
✅ 支持中文 - 完全支持中文标签名称

## 常见问题

### Q: 添加新标签后需要做什么？
A: 只需重新构建网站（`bundle exec jekyll build`），插件会自动生成新的标签页面。

### Q: 可以手动修改标签页面吗？
A: 可以。如果你在 `_tags_page/` 中手动编辑了标签页面，插件不会覆盖它。

### Q: 删除标签后会怎样？
A: 对应的标签页面文件不会自动删除，你可以手动删除。
